Hexo文件结构与常用命令

Hexo 文件结构

Hexo初始化后生成以下文件：

  • .github、.gitignore：从GitHub拉下来时自带的，我们不会推送Hexo的源码，因此这两项可以删除。

  • _config.landscape.yml：大概是默认主题(landscape)的配置文件？也可以删除。

  • node_modules：node模块库，为hexo安装的插件都会安装在该文件夹下。

  • scaffolds：模板/布局文件夹，当新建文章时，可以根据 scaffolds 中的模板/布局来创建文件。Hexo 的模板/布局是指在新建的 markdown 文件中默认填充的内容。例如，如果修改 scaffolds/post.md 中的 Front-matter 内容，那么每次新建一篇文章时都会包含这个修改。Hexo 有三种默认模板/布局：post、page 和 draft，它们分别对应不同的路径，你可以自定义其他模板/布局。

    模板/布局 保存路径
    post source/_posts
    page source
    draft source/_drafts
    自定义模板 source/_posts

  • source：资源文件夹，存放用户资源。所有 hexo new 的 markdown 文件都会存放在该目录下。但除 _posts 文件夹之外，开头命名为 _ (下划线)的文件/文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

  • themes：主题文件夹。默认主题与下载的主题存在该文件夹。

  • _config.yml：站点配置文件，在此可以配置大部分参数。另外的参数需要在 themes 文件夹下的主题文件夹内的 _config.yml 文件配置，我们称之为主题配置文件，它们是不同的配置文件。

  • package.json：包版本信息，无需关注。

  • package-lock.json：hexo init 时自动生成，锁定安装时的包的版本号，无需关注。

Front-matter

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量。新建文章的默认Front-matter在scaffolds 文件夹的相应文件修改。

以下是 Hexo 的预定义参数，可以在模板和文章中使用这些参数。但不同主题可能有不同的额外参数。

参数 描述 默认值
layout 模板/布局
title 标题
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于page布局）
categories 分类（不适用于page布局）
permalink 重写文章的永久链接
type 指定页面类型，可设类型为”tags”或”categories”或不设定 默认不设定，此时为普通页面
