深度学习多目标跟踪理论与实战

发表于2022-09-25|更新于2022-09-25|深度学习多目标跟踪
|字数总计:2.6k|阅读时长:8分钟|阅读量:

什么是多目标跟踪

以普遍理论而言，多目标跟踪就是在一段视频里得到感兴趣的物体的运动轨迹。

这个任务的输入是视频帧，输出即是物体的运动轨迹（包括该物体的位置(检测框)、类别、编号）

从网络模型来分，多目标跟踪方法可以分为两种。

  • Two-step，需要使用两个模型完成多目标跟踪，先检测，后跟踪，Tracking by detection。
  • One-shot，只需要一个One-shot模型，可以同时得到每个物体的检测框和他们的ReID特征

多目标跟踪的评价指标

  • MOTA：比较主要的指标，也就是精度，根据fp、fn和ID跳变的次数来得到的，因为ID跳变次数的数量级比fp、fn要小很多，所以MOTA比较偏向于检测的性能
  • IDF1：主要计算一条轨迹上ID正确的F1的值，比较偏向于跟踪的性能
  • IDs：ID跳变的次数，越小越好
  • TPR：模型的ReID的性能

Two-step：DeepSort (经典)

经典Two-step算法，Tracking by detection。这种使用比较广、比较简单的范式分为两步，第一步是通过object detection得到位置(检测框)、类别，第二步是使用data association将不同帧之间的检测框连接起来，得到他们的ID(编号)。

因为是Tracking by detection，第一步的检测器直接使用现成的，所以它的主要工作在第二步的data association部分。

Data association

Similarity

主要用来计算当前帧的检测框和上一帧的相似度，主要包含三个部分。

  • Motioin：运动相似度，代表方法是Kalman filter(卡尔曼滤波)，通过框在上一帧的位置来预测下一帧的位置。
  • Location：位置相似度，代表方法是计算框与框的IoU。
  • Appearance：外观相似度，代表方法是使用 ReID model 提取每个框的特征，计算ReID特征间的Cosine距离

Assignment

得到相似度后，根据相似度结果进行Assignment。Assignment可以理解为ID的分配过程。代表方法是匈牙利算法，根据跟踪片段和检测框之间两两的相似度用整体相似度最高的方法完成连接。

One-shot：FairMOT (IJCV 2021)

发表于IJCV 2021，一个比较新的One-shot方法。

Motivation

  • 目标检测 和 行人重识别都是MOT非常重要的部分
  • One-shot 方法在速度上比 Two-step方法更有优势，因为它在backbone分享了大量特征。
  • One-shot 方法的ReID分支一般来说是比较难学的，他会受到检测分支的影响
  • 重新思考 目标检测 和 行人重识别 之间的关系，并公平的对待他们。主要来说就是如何在一个One-shot方法里面让 检测 和 行人重识别 两个分支都学习的比较好

之前的One-shot方法的不足

  • 基于Anchors的方法会带来一些不公平性
    • ReID任务通常会被忽视，比如fast-rcnn中先进行rpn，偏重检测，ReID是后面再来算的
    • 一个Anchors中可能会包含多个物体
    • 同一个物体可能会有多个Anchors对应它，导致它的外观学习比较困难
    • 解决方法：每个物体只取中心点周围的特征
  • 特征的提取会带来一些不公平性
    • 目标检测需要比较深层的特征，例如要检测人，那么它需要去寻找不同的人之间的共同的特征
    • ReID需要比较浅层的特征，它的目的是区分不同的人，需要去寻找不同的人之间的差异性的特征
    • 实验发现多层特征融合是可以缓解这种特征冲突的，因为它可以让两个任务从各自需要的地方提取特征
  • 特征的维度会带来一些不公平性
    • 高维的特征(例如2048或1024维)会损伤目标检测的精度
    • 和一般的ReID任务不同，MOT中的ReID任务并不需要那么高纬度的ReID特征(MOT任务比较简单)
    • 较低纬度的特征需要的计算量更少，推理速度更快

网络结构

  • 整体网络结构：输入图像通过 encoder-decoder 的backbone，输出一个比较高维(高分辨率)的特征，到两个平行的分支（detection和ReID）

  • Encoder-Decoder：

  • Detection分为三个部分：

    • heatmap：学习目标的中心点
    • box size：学习目标的宽和高
    • center offset：学习中心点因为四倍下采样带来的小数点的偏移

  • ReID主要是在全图的特征中取到每个物体的中心点对应的特征，作为该物体的特征

Data association

主要分为两步：

  • Motioin + Appearance：运动相似度+外观相似度，卡尔曼滤波 和 ReID特征间的Cosine距离

    得到这两个相似度后对他们进行加权求和得到总相似度。使用这个总相似度直接输入匈牙利算法做一次匹配。

  • 对于没有匹配上的检测框，他们的ReID特征已经不可靠了(大多是遮挡情况)，此时需要使用位置相似度(IoU)进行第二次匹配，把被遮挡的物体跟踪回来

ByteTrack (ECCV 2022)

发表于ECCV 2022，也是实战推荐使用的算法。

Motivation

之前的方法(几乎所有的)，为了滤除背景，在跟踪时只保留置信度高于阈值的框，置信度较低的框都是直接丢掉。然而置信度较低的检测框中往往会包含一些正确的物体，比如被遮挡的物体，这会带来不可逆转的漏检和碎片轨迹。

Contribution 主要贡献

  • 提出了由检测驱动的方法BYTE，通过跟踪片段和检测框之间的相似度来从低分的检测框中把被遮挡的物体找出来，并且能有效过滤背景
  • 这个数据关联的方法BYTE可以用在很多不同的跟踪任务中，对于MOTA或IDF1有1~10个点的提升，它的泛化性能非常强
  • 设计了一个简单且性能非常好的方法ByteTrack，第一次以30FPS的速度达到了80的MOTA

ByteTrack: Data association

作为一种Two-step方法，检测器使用了旷视的YOLOX，因此它的主要工作也在Data association。

  • Motioin：在大部分数据集上只需要使用卡尔曼滤波计算以下运动相似度即可

  • Appearance：只有在一些难度很大的数据集(BDD100K)上才需要使用ReID来计算外观相似度
    （也就是说，如果你的使用场景比较简单的话，甚至不需要训练ReID模型，撸个算法就够了！）

ByteTrack 算法流程

  • 输入：一段视频V, 目标检测器Det, 置信度阈值Thld.
  • 输出：对于视频V的跟踪片段T
  • 初始化：跟踪片段T置0
  • 遍历输入视频V的每一帧
    • 通过检测器得到当前帧的所有检测框
    • 将这些检测框分为高分组和低分组
    • Similarity#1：上一帧所有跟踪片段 和 高分的检测框 计算相似度
    • Assignment#1。（匈牙利算法）
    • Similarity#2：第一次匹配没有匹配上的跟踪片段 与 低分的检测框 计算相似度
    • Assignment#2。
    • 对于两次都匹配不到的检测框：保留30帧，仍然匹配不到就丢弃
  • 结束。

ByteTrack 的优点与局限性

  • 对于开始置信度较高，中途置信度变低(被遮挡) 的物体，ByteTrack 效果很好。

  • 假如一个物体一开始的置信度就比较低，那么它基本就匹配不到了
    （但你需要优化的是检测器 :) ，关我ByteTrack什么事？）

不同场景下的推荐方法

使用ByteTrack在不同场景下，Similarity部分的推荐方法

固定摄像头场景

  • 数据集：MOT17，MOT20，HiEve

  • 对于这种比较简单的场景，一般Similarity单独使用卡尔曼滤波或IoU就足够了

运动相机场景 (自动驾驶)

  • 数据集：KITTI，BDD100K，Waymo，MOT17部分
  • 对于这种场景只有卡尔曼滤波是不够的，因为运动相似度会受到相机的运动的影响
  • 给卡尔曼滤波加上相机运动补偿：ECC，ORB，SuperPoint
  • 对于更复杂的场景，加上ReID方法：FastReID，UniTrack
  • UniTrack是在ImageNet上训练好的分类模型，支持不同的类别，大部分情况下不用训练，可以直接使用

低帧率 (帧间差异大)

  • 数据集：KITTI，BDD100K
  • 此时卡尔曼滤波效果变得较差
  • 代替卡尔曼滤波（可学习的模型）：CenterTrack，MOTR (基于Transformer)，P3AFormer
  • ReID：QDTrack

统一的外观和多样的运动

  • 数据集：DanceTrack
  • 经典场景是识别舞台上的舞者，他们一般统一着装，并且舞蹈动作比较剧烈。
  • 此时卡尔曼滤波和ReID的方法效果都比较差
  • 可能的方法：MOTR (基于Transformer)，OC-SORT (非线性Motioin模型）

总结

  • 物体运动速度适中、干扰小 (固定摄像头)：两次计算相似度都直接采用IoU即可
  • 物体运动速度飞快 (运动相机、低帧率)：第一次最好IoU+ReID加权计算，第二次仍采用IoU
  • 物体不仅跑的飞快而且运动轨迹乱七八糟：两次都使用可学习模型代替IoU，例如MOTR、OC-SORT

ByteTrack实战：人流量统计

任务分析

  • 固定摄像头下，相机无运动，帧率适中(24或30帧)。
  • 识别目标：街道上的行人。因此检测器可以直接使用各大平台发布的预训练模型。
  • 目标轨迹：逛街中的行人一般不会释放闪现、左右横跳等增加跟踪难度的技能，因此不需要使用ReID模型。
  • 要求：(均为实时显示)
    • 显示当前画面中的人数：有ID的结果数相加即可
    • 绘出每个人的运动轨迹：
    • 显示历史同屏人数峰值
    • 显示历史人数统计：一个人至少在画面中出现1秒才计入统计数据
