C++面向对象语法

类基础

构造函数

初始化对象。参数可有可无，无返回值，可以重载。

析构函数

销毁对象。无参数，无返回值，（可以自动释放基础变量）

常成员函数 const

只读不写。防止意外修改成员变量

静态成员 static

依赖于类，不依赖于某个对象。典型应用：统计某个类共创建了多少个对象。

类定义标准

  • 方法（函数）放在 public
  • 属性（变量）放在 private
  • 要修改属性，应该通过方法去修改，且方法中应有检验机制，满足条件才允许修改

类的派生和继承

基本概念：

  • 学生类可以细分为本科生、研究生
  • 本科生、研究生都是学生，是学生的子类
  • 学生是父类，又称基类或超类
  • 所谓派生，是相对于父类而言的。父类派生子类，基类派生派生类
  • 所谓继承，是相对于子类而言的。子类从父类继承

继承规则

  • 访问属性关键字：public（公有）protect（保护）private（私有）
  • 公有继承时，基类的公有成员和保护成员被派生类继承后，访问属性不变
  • 保护继承中，基类的公有成员和保护成员被派生类继承后变成保护成员
  • 私有继承时，基类的公有成员和保护成员被派生类继承后变成私有成员
  • 无论哪种继承，子类/派生类 都无法访问或继承 父类/基类 的私有成员

子类的构造函数

  • 子类可以没有构造函数，但无论子类有没有构造函数，创建子类对象时，父类的构造函数都会被优先调用。先运行父类的构造，再是子类

  • 子类无法继承父类的构造、析构函数，只能调用。（子类与父类的属性不相同）

  • 带参数的子类构造函数

    程序会先调用父类student的构造函数，把 25, “张三”，两个父类需要的值传入父类带参数的构造函数，再把 “计算机系” 赋值给子类自己的other属性。完成全动态初始化赋值。

多态（多态基于继承）

基本概念

加法是专属与数字类的，对于字符串，显然是不能使用加法的。
然而，string类早已实现了两string相加，这是如何实现的？
这就是多态。核心思想是不同的类，使用相同的方法。

相同函数名执行不同的内容的3中方法：

重载（overload）一个类中定义多个同名方法。

  • 重载的特征：
    函数名相同，参数列表必须不同，必须位于同一个 域/类 中。
    与访问修饰符、返回值类型、抛出异常类型无关。
  • 执行时，判断给定参数列表来确定要执行那个方法。

覆盖（override）子类方法覆盖父类方法。（多态）

  • 覆盖的特征：
    1.函数名、参数列表、返回值必须完全相同
    2.父类中要被覆盖的方法必须带有virtual关键字（是虚函数）
    3.子类方法的访问修饰符权限不能比父类中要被覆盖的方法低
    4.子类方法不能抛出比父类方法更多的异常。(一致、是其子集、不抛出异常)
    5.被覆盖的方法不能是final类型。final修饰的方法无法被覆盖
    6.被覆盖的方法不能为private。
    7.被覆盖的方法不能为static。即使父类和子类中的方法都是静态的，并且满足覆盖条件，但是仍然不会发生覆盖
  • 执行时，父类对象调用父类内的方法，子类对象调用子类内的方法

隐藏（hide）子类中的方法隐藏（屏蔽）父类中的同名方法

  • 理解：在调用一个类的成员函数的时候,编译器会沿着类的继承链逐级的向上查找函数的定义,如果找到了那么就停止查找了,所以如果一个派生类和一个基类都有同一个同名(暂且不论参数是否相同)的函数,而编译器最终选择了在派生类中的函数,那么我们就说这个派生类的成员函数”隐藏”了基类的成员函数,也就是说它阻止了编译器继续向上查找函数的定义….
  • 隐藏的特征：函数名相同、分别位于子类和父类中。
  • 覆盖与隐藏的区别：
    1.参数列表不同：不构成覆盖关系，构成隐藏关系
    2.参数列表相同：是否有virtual关键字。有则覆盖，无则隐藏。

类指针

基本规则（与结构体指针相同）

考虑继承（省略类定义，只考虑公有继承）

父类指针可以指向子类对象，因为子类是继承自父类的。
但这样不可以调用子类的属性和方法，还是只能调用父类的属性和方法。

虚函数（为多态服务）

多态语法

c++多态的核心：只要在程序开始设置一个父类指针，之后这个指针就可以动态的指向不同的类，以调用不同类的方法，从而实现不同数据类使用相同的方法。

重载是编译时决定，多态是运行时决定。

重载在编译时，把两个同名函数当作不同函数处理，编译阶段已经确定了要调用的是哪个函数。
多态在编译时，无法确定要调用的是哪个函数，只好在内存中建立并维护一个虚函数表，将所有虚函数都放在里面，运行时再根据指针确定调用哪个函数。（覆盖必须是虚函数的原因）

抽象类与纯虚函数

  • 概念理解：
    学生是一个类，研究生/本科生/中小学生，都是学生类的子类。
    只属于学生类，却不属于学生类的任何一个子类的情况，是不存在的。
    学生类不应该存在自己的对象，于是学生类是一个抽象类
    （实际程序中，抽象类不能，也不应该创建自己的对象）

  • 语法：（抽象类 必须声明至少一个 纯虚函数）

  • 抽象类拥有一些共有的属性和方法，例如年龄姓名，但不能创建自己的对象，创建对象依赖子类实现。

c++与新兴高级编程语言对比

  • 封装、继承、多态是面向对象的三大法宝，也是所有编程语言通用的。
  • c++的多态依赖于指针实现，而新兴语言(java/python/verilog)最大的革新就是取消了指针，可以说是解放了程序员。那他们是怎么实现多态的呢？
  • 虽然指针被隐藏了，但只要还使用内存，指针就必须存在。在高级语言中，取消指针的本质就是将变量名于指针融为一体，变量名本身就是一个指针。当你写下 a = 10 ，本质上a本身是一个指针，指向内存中的一块地址，这个地址存放了数值10。再次对a进行赋值，例如 a = 15 ，此时a这个指针指向了一块新地址，而原来的数值10依然存放于内存中的老地址，没有被复写掉。
  • 因此高级语言赋值的本质，就是赋指针。当指针就是变量名，实现多态也就变的非常方便，系统根据传入的变量名(本质还是指针)即可判断调用哪个函数。这就是 字符串 a + b 在python中是多态，在c++中是重载的原因。
  • python中无需定义数据类型的本质：先赋值 a = 10 再赋值 a = "text" ，即前后赋值不同的数据类型不会出错的原因是变量名的本质是指针，而指针可以随便指向不同的地址，自然不会出错。
