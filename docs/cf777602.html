在 Jetson 上安装 mmdeploy:0.9.0

发表于2022-10-17|更新于2022-11-09|深度学习模型部署
|字数总计:1.8k|阅读时长:7分钟|阅读量:

首先需要在 Jetson 模组上安装 JetPack SDK。
此外，在利用 MMDeploy 的 Model Converter 转换 PyTorch 模型为 ONNX 模型时，需要创建一个装有 PyTorch 的环境。
最后，关于编译工具链，要求 CMake 和 GCC 的版本分别不低于 3.14 和 7.0。

配置安装环境(docker)

MMDeploy 中的 Model Converter 依赖于 MMCV 和推理引擎 TensorRT。

检查 JetPack SDK 版本

此文写于 mmdeploy_0.9.0 发布后，由官方文档魔改而来。这里选择 JetPack 4.6.1 。

JetPack SDK 4+ 自带 python 3.6。切勿尝试升级 python 版本，会破坏 JetPack 环境。

此外，如果你的 Jetson 部署时可以选择安装 JetPack 5+，那么你应使用官方发布的一键搭建环境的脚本，而不是使用此文的方式。

配置 TensorRT

JetPack SDK 自带 TensorRT。

安装 Model Converter 的依赖项

  • 安装 MMCV

    MMCV 还未提供针对 Jetson 平台的预编译包，因此我们需要从源对其进行编译。

  • 安装 ONNX

  • 安装 h5py 和 pycuda

    Model Converter 使用 HDF5 存储 TensorRT INT8 量化的校准数据；需要 pycuda 拷贝显存

安装 MMDeploy

安装 Model Converter

由于一些算子采用的是 OpenMMLab 代码库中的实现，并不被 TenorRT 支持，
因此我们需要自定义 TensorRT 插件，例如 roi_align， scatternd 等。
你可以从这里找到完整的自定义插件列表。

检查安装成果

tools/check_env.py 将会打印mmdeploy安装时的环境信息。

重点关注以下内容：

  • 环境信息：cuda是否可用、显卡型号是否正确
  • 后端信息：Jetson 主要利用TensorRT后端，因此只关注TensorRT。版本是否正确、算子是否可用。

备份docker镜像

确保检查环境信息通过，没有问题再进行这一步。

  • 安装转模型必要的包

  • 退出docker容器，将docker容器保存成tar包

  • 从 tar 包导入docker镜像

安装 C/C++ Inference SDK

如果你不需要使用 MMDeploy C/C++ Inference SDK 则可以跳过本步骤。

MMDeploy 的 C/C++ Inference SDK 依赖于 spdlog， OpenCV， ppl.cv 和 TensorRT 等。

安装 SDK 的依赖项

  • 安装 spdlog

    “spdlog 是一个快速的，仅有头文件的 C++ 日志库。”

  • 安装 ppl.cv

    “ppl.cv 是 OpenPPL 的高性能图像处理库。”

TroubleShooting

安装

  • pip install 报错 Illegal instruction (core dumped)

    如果上述方法仍无法解决问题，检查是否正在使用镜像文件。如果是的，可尝试：

转模型

  • #assertion/root/workspace/mmdeploy/csrc/backend_ops/tensorrt/batched_nms/trt_batched_nms.cpp,98 or pre_top_k need to be reduced for devices with arch 7.2

    1. 设置为 MAX N 模式并执行 sudo nvpmodel -m 0 && sudo jetson_clocks。
    2. 效仿 mmdet pre_top_k，减少配置文件中 pre_top_k 的个数，例如 1000。
    3. 重新进行模型转换并重新运行 demo。

  • 报错：OSError: Could not find lib geos_c or load any of its variants ['libgeos_c.so.1', 'libgeos_c.so'].

    解决：apt-get install libgeos-dev

  • 报错：无法加载xxxx算子

    解决：重新编译 TensorRT 自定义算子并安装 model converter。

  • 报错：Error Code 10: Internal Error (Could not find any implementation for node xxxx.)

    解决：暂无解决办法，但可以使用老版本环境进行转模型，老版本没有这个问题
